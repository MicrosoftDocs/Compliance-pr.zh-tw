## <a name="integrity-checks"></a><span data-ttu-id="b2b51-172">完整性檢查</span><span class="sxs-lookup"><span data-stu-id="b2b51-172">Integrity Checks</span></span>

<span data-ttu-id="b2b51-173">SharePoint 會使用各種方法來確保 blob 和中繼資料的完整性，在資料生命週期的各階段：</span><span class="sxs-lookup"><span data-stu-id="b2b51-173">SharePoint uses various methods to ensure the integrity of blobs and metadata at all stages of the data lifecycle:</span></span>

- <span data-ttu-id="b2b51-174">**儲存在中繼資料中的檔案雜湊**：整個檔案的雜湊會以檔案中繼資料儲存，以確保在所有作業期間維護檔層級的資料完整性</span><span class="sxs-lookup"><span data-stu-id="b2b51-174">**File hash stored in metadata**: Hash of the entire file is stored with file metadata to ensure document level data integrity is maintained during all operations</span></span>
- <span data-ttu-id="b2b51-175">**Blob 雜湊儲存于中繼資料中**：每個 Blob 專案會儲存加密內容的雜湊，以防止基礎 Azure 儲存區損毀。</span><span class="sxs-lookup"><span data-stu-id="b2b51-175">**Blob hash stored in metadata**: Each blob-item stores a hash of the encrypted content to protect against corruption in underlying Azure storage.</span></span>
- <span data-ttu-id="b2b51-176">**資料完整性工作**：每隔14天，會列出資料庫中的專案，並符合 Azure 儲存體中列出的 blob，針對每個網站進行完整性掃描。</span><span class="sxs-lookup"><span data-stu-id="b2b51-176">**Data integrity job**: Every 14 days, each site is scanned for integrity by listing items in the database and matching those up with listed blobs in Azure storage.</span></span> <span data-ttu-id="b2b51-177">工作會報告任何 blob 參照缺失的儲存體-blob，而且可以視需要透過 [Azure 儲存體虛刪除](https://docs.microsoft.com/azure/storage/blobs/soft-delete-blob-overview) 功能取得這些 blob。</span><span class="sxs-lookup"><span data-stu-id="b2b51-177">The job reports any blob-references missing storage-blobs and can retrieve those blobs through the [Azure storage soft-delete](https://docs.microsoft.com/azure/storage/blobs/soft-delete-blob-overview) feature if needed.</span></span>
