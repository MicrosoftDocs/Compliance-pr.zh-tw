## <a name="how-does-microsoft-365-protect-audit-logs"></a><span data-ttu-id="ef522-130">Microsoft 365 如何保護審核記錄？</span><span class="sxs-lookup"><span data-stu-id="ef522-130">How does Microsoft 365 protect audit logs?</span></span>

<span data-ttu-id="ef522-131">Microsoft 365 用來收集和處理審計記錄的工具不允許對原始的審計記錄內容或時間順序進行永久或不可逆的變更。</span><span class="sxs-lookup"><span data-stu-id="ef522-131">The tools used in Microsoft 365 to collect and process audit records do not allow permanent or irreversible changes to the original audit record content or time ordering.</span></span> <span data-ttu-id="ef522-132">Cosmos 中儲存的 Microsoft 365 資料存取權僅限於授權的人員。</span><span class="sxs-lookup"><span data-stu-id="ef522-132">Access to Microsoft 365 data stored in Cosmos is restricted to authorized personnel.</span></span> <span data-ttu-id="ef522-133">此外，Microsoft 365 會將審核記錄的管理限制為負責進行審計功能的安全小組成員的有限子集。</span><span class="sxs-lookup"><span data-stu-id="ef522-133">In addition, Microsoft 365 restricts the management of audit logs to a limited subset of Security team members responsible for audit functionality.</span></span> <span data-ttu-id="ef522-134">安全小組不具備 Cosmos 的管理存取權。</span><span class="sxs-lookup"><span data-stu-id="ef522-134">The Security Team does not have standing administrative access to Cosmos.</span></span> <span data-ttu-id="ef522-135">系統管理存取需要即時 (JIT) 存取權，而且會記錄及審核 Cosmos 對記錄機制所做的所有變更。</span><span class="sxs-lookup"><span data-stu-id="ef522-135">Administrative access requires Just-In-Time (JIT) access approval, and all changes to logging mechanisms for Cosmos are recorded and audited.</span></span> <span data-ttu-id="ef522-136">審核記錄會保留足夠長的時間，以支援事件調查並符合法規需求。</span><span class="sxs-lookup"><span data-stu-id="ef522-136">Audit logs are retained long enough to support incident investigations and meet regulatory requirements.</span></span> <span data-ttu-id="ef522-137">Cosmos 中的審計記錄資料保留量的確切期限是由服務小組決定。大部分的審計記錄資料會保留90天或更長的時間。</span><span class="sxs-lookup"><span data-stu-id="ef522-137">The exact period of audit log data retention in Cosmos is determined by the service teams; most audit log data is retained for 90 days or longer.</span></span>

<span data-ttu-id="ef522-139">在上傳記錄資料之前，ODL 應用程式會使用清理服務來移除任何包含客戶資料的欄位，例如承租人資訊和使用者識別資訊，並以雜湊值取代這些欄位。</span><span class="sxs-lookup"><span data-stu-id="ef522-139">Prior to uploading log data, the ODL application uses a scrubbing service to remove any fields that contain customer data, such as tenant information and end-user identifiable information, and replace those fields with a hash value.</span></span> <span data-ttu-id="ef522-140">匿名和雜湊記錄會重新寫入並上傳至 Cosmos。</span><span class="sxs-lookup"><span data-stu-id="ef522-140">The anonymized and hashed logs are rewritten and then uploaded into Cosmos.</span></span> <span data-ttu-id="ef522-141">所有的記錄傳送都會透過 TLS 加密的連線進行， (FIPS 140-2) 。</span><span class="sxs-lookup"><span data-stu-id="ef522-141">All log transfers occur over a TLS encrypted connection (FIPS 140-2).</span></span>
