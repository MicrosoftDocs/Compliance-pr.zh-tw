## <a name="how-does-microsoft-365-ensure-isolation-between-customer-tenants"></a><span data-ttu-id="4e519-114">Microsoft 365 如何確保客戶承租人間的隔離？</span><span class="sxs-lookup"><span data-stu-id="4e519-114">How does Microsoft 365 ensure isolation between customer tenants?</span></span>

<span data-ttu-id="4e519-115">Microsoft 的雲端服務是以假設所有租使用者可能會敵意所有其他租使用者所組成。</span><span class="sxs-lookup"><span data-stu-id="4e519-115">Microsoft's cloud services are built on the assumption that all tenants are potentially hostile to all other tenants.</span></span> <span data-ttu-id="4e519-116">為了正確地將承租人彼此隔離，Microsoft 會執行各種隔離技術和控制措施。</span><span class="sxs-lookup"><span data-stu-id="4e519-116">To properly isolate tenants from one another, Microsoft implements a variety of isolation technologies and controls.</span></span> <span data-ttu-id="4e519-117">這些控制項的設計目的是為了避免資訊洩露或未經授權存取跨承租人的客戶資料，並防止一個承租人的動作對其他租使用者的服務造成不良影響。</span><span class="sxs-lookup"><span data-stu-id="4e519-117">These controls are designed to safeguard against information leakage or unauthorized access to customer data across tenants and to prevent the actions of one tenant from adversely affecting the service for another tenant.</span></span>

<span data-ttu-id="4e519-118">使用 Azure Active Directory (Azure AD) ，以邏輯方式隔離 Microsoft 365 承租人內的客戶內容。</span><span class="sxs-lookup"><span data-stu-id="4e519-118">Customer content is logically isolated within Microsoft 365 tenants using Azure Active Directory (Azure AD).</span></span> <span data-ttu-id="4e519-119">Microsoft 365 中的使用者驗證不僅會驗證使用者身分識別，還會驗證使用者帳戶所屬的承租人身分識別，以防止使用者存取其租使用者環境以外的資料。</span><span class="sxs-lookup"><span data-stu-id="4e519-119">User authentication in Microsoft 365 verifies not only the user identity, but also the tenant identity the user account is part of, preventing users from accessing data outside their tenant environment.</span></span> <span data-ttu-id="4e519-120">若要對 Azure AD 的邏輯隔離進行補充，客戶內容會永遠在靜止時加密，並可在傳輸中加密。</span><span class="sxs-lookup"><span data-stu-id="4e519-120">To supplement the logical isolation of Azure AD, customer content is always encrypted at rest and in transit.</span></span> <span data-ttu-id="4e519-121">個別服務也可能會提供額外的租使用者隔離層，例如在個別的加密資料庫中 SharePoint 線上隔離租使用者資料。</span><span class="sxs-lookup"><span data-stu-id="4e519-121">Individual services may also provide additional layers of tenant isolation, such as SharePoint Online isolation of tenant data in separate, encrypted databases.</span></span>

## <a name="how-does-microsoft-365-engineer-resilient-services-that-avoid-single-points-of-failure"></a><span data-ttu-id="4e519-122">Microsoft 365 工程師如何彈性服務以避免單點失敗？</span><span class="sxs-lookup"><span data-stu-id="4e519-122">How does Microsoft 365 engineer resilient services that avoid single points of failure?</span></span>

<span data-ttu-id="4e519-123">Microsoft 設計並組建雲端服務，以最大限度地提升可靠性，並將對客戶面臨的負面影響降至最低，以應對一般作業的錯誤和挑戰。</span><span class="sxs-lookup"><span data-stu-id="4e519-123">Microsoft designs and builds cloud services to maximize reliability and minimize the negative effects on customers in the face of faults and challenges to normal operations.</span></span> <span data-ttu-id="4e519-124">此策略的開始是連接地理位置分散的資料中心網路的網路設計。</span><span class="sxs-lookup"><span data-stu-id="4e519-124">This strategy begins with the design of the network connecting our geographically distributed datacenters.</span></span> <span data-ttu-id="4e519-125">Microsoft 的網路架構包括直接互連和多個網路路徑。</span><span class="sxs-lookup"><span data-stu-id="4e519-125">Microsoft's network architecture includes direct interconnections and multiple network paths.</span></span> <span data-ttu-id="4e519-126">Microsoft 365 服務利用此冗余，自動路由傳送失敗的流量，以提升服務品質。</span><span class="sxs-lookup"><span data-stu-id="4e519-126">Microsoft 365 services leverage this redundancy to automatically route traffic around failures to improve service quality.</span></span>

<span data-ttu-id="4e519-127">在服務層級，Microsoft 365 的修復原則會將軟體恢復優先順序。</span><span class="sxs-lookup"><span data-stu-id="4e519-127">At the service level, Microsoft 365's resilience strategy prioritizes software resiliency.</span></span> <span data-ttu-id="4e519-128">在任何情況下，我們的服務都會以自動化服務狀況監視方式部署在主動/主動設定中，允許服務偵測和復原許多常見的錯誤和失敗，而不需要人工干預。</span><span class="sxs-lookup"><span data-stu-id="4e519-128">Wherever possible, our services are deployed in active/active configurations with automated service health monitoring, allowing the service to detect and recover from many common faults and failures without human intervention.</span></span> <span data-ttu-id="4e519-129">除了主動/主動設定之外，Microsoft 365 服務會確保服務部署在不同的容錯區域中，以避免一個區域中的錯誤影響其他區域的可用性，以提升容錯能力。</span><span class="sxs-lookup"><span data-stu-id="4e519-129">In addition to active/active configurations, Microsoft 365 services increase fault-tolerance by ensuring the service is deployed in separate fault zones, preventing a fault in one zone from affecting the availability of other zones.</span></span>

<span data-ttu-id="4e519-130">資料恢復功能會保護 Microsoft 365 服務中的資料完整性及可用性，以補充服務彈性。</span><span class="sxs-lookup"><span data-stu-id="4e519-130">Data resiliency complements service resiliency by protecting the integrity and availability of data in Microsoft 365 services.</span></span> <span data-ttu-id="4e519-131">我們的服務使用本機儲存裝置冗余和地域冗余，將客戶資料的複本複寫到不同的容錯區域。</span><span class="sxs-lookup"><span data-stu-id="4e519-131">Our services use local storage redundancy and geo-redundancy to replicate copies of customer data into different fault zones.</span></span> <span data-ttu-id="4e519-132">如果一個容錯區域中的資料損毀或遺失，可在另一個容錯區域中存取，而不會失去可用性。</span><span class="sxs-lookup"><span data-stu-id="4e519-132">If data is corrupted or lost in one fault zone, it can be accessed in another fault zone without loss of availability.</span></span> <span data-ttu-id="4e519-133">自動完整性檢查會自動還原由許多實體或邏輯損毀所影響的資料。</span><span class="sxs-lookup"><span data-stu-id="4e519-133">Automated integrity checking automatically restores data impacted by many kinds of physical or logical corruption.</span></span> <span data-ttu-id="4e519-134">Microsoft 365 也會為客戶提供工具，以在 Exchange Online 中與客戶意外刪除或修改資料，並 SharePoint 線上。</span><span class="sxs-lookup"><span data-stu-id="4e519-134">Microsoft 365 also provides customers with tools to restore data accidentally deleted or modified by the customer in Exchange Online and SharePoint Online.</span></span>

<span data-ttu-id="4e519-136">Microsoft 365 服務小組會識別重要的系統元件及其相依性，以作為商務持續性管理的一部分。</span><span class="sxs-lookup"><span data-stu-id="4e519-136">Microsoft 365 service teams identify critical system components and their dependencies as part of Business Continuity Management.</span></span> <span data-ttu-id="4e519-137">此外，Microsoft 365 檔和追蹤所有外部系統連線，以確保網路防火牆設定只允許授權的連線。</span><span class="sxs-lookup"><span data-stu-id="4e519-137">In addition, Microsoft 365 documents and tracks all external system connections to ensure only authorized connections are allowed in network firewall configurations.</span></span> <span data-ttu-id="4e519-138">Microsoft 365 系統、相依性及外部連線會記錄在 Microsoft 365 的資訊安全性架構中。</span><span class="sxs-lookup"><span data-stu-id="4e519-138">Microsoft 365 systems, dependencies, and external connections are documented in Microsoft 365's information security architecture.</span></span> <span data-ttu-id="4e519-139">資訊安全架構和對應的資料流程圖都會至少每年進行檢查及更新，而且每當對系統進行重大變更時，也會進行更新。</span><span class="sxs-lookup"><span data-stu-id="4e519-139">Both the information security architecture and corresponding data flow diagrams are reviewed and updated annually at a minimum, as well as whenever significant changes are made to the system.</span></span>

<span data-ttu-id="4e519-140">Microsoft 365 架構會定期進行驗證，並自動使用雲端架構工具來驗證與安全性原則的搭配，並持續測試隔離和恢復功能。</span><span class="sxs-lookup"><span data-stu-id="4e519-140">Microsoft 365 architecture is validated regularly and automatically using cloud-based tools to verify alignment with our security principles and to continuously test isolation and resiliency features.</span></span> <span data-ttu-id="4e519-141">架構驗證可讓您自動識別目前服務的目前狀態已正軌至所需狀態的實例，並會標記出任何偏差以進行審閱與緩解。</span><span class="sxs-lookup"><span data-stu-id="4e519-141">Architectural validation works to automatically identify instances where the current state of the service has drifted from the desired state, flagging any deviations for review and mitigation.</span></span> <span data-ttu-id="4e519-142">架構驗證的目標是確保服務基礎結構的安全性功能仍可如預期的方式運作。</span><span class="sxs-lookup"><span data-stu-id="4e519-142">The goal of architecture validation is to ensure the security capabilities of our service infrastructure continue to function as expected.</span></span>
